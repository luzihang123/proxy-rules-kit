# 项目技术与语法教学

这份文档面向第一次维护 RouteKit 的人，说明本项目主要用到的技术、配置语法和公开仓库维护边界。

## 项目定位

RouteKit 是公开模板仓库，只保存可复用的分流规则、策略组和客户端适配示例。真实节点订阅 URL、机场 token、公司域名、家庭网络地址、本机路径和私有 Gist 都不应该写进本仓库。

## 主要技术

- YAML：Clash Verge Rev、Clash Party、OpenClash、Clash for Windows 的策略组和规则模板。
- JavaScript：Clash Verge Rev / Clash Party 的订阅扩展脚本，在刷新订阅后动态注入规则和策略组。
- Shadowrocket 配置语法：`routekit.conf` 使用 `[General]`、`[Proxy Group]`、`[Rule]` 三段式配置。
- 规则列表：`.list` / `.rules` 文件按一行一条规则维护，便于不同客户端复用。
- 正则表达式：用地区关键词匹配节点名，例如 `香港`、`HK`、`Japan`、`JP`、`Australia`、`AU`。

## 常见规则语法

```text
DOMAIN-SUFFIX,example.com,策略组名
DOMAIN,api.example.com,策略组名
DOMAIN-KEYWORD,keyword,策略组名
MATCH,兜底策略组
```

含义：

- `DOMAIN-SUFFIX` 匹配域名后缀，适合整站分流。
- `DOMAIN` 只匹配完整域名，适合单个 API 域名。
- `DOMAIN-KEYWORD` 按关键词匹配，命中范围更大，新增时要谨慎。
- `MATCH` 是兜底规则，通常放在最后。

## 策略组维护方法

策略组名必须和规则里的目标名完全一致。比如规则写 `📈 RouteKit-Broker`，客户端配置里也必须存在同名策略组，否则会出现类似 `proxy [📈 RouteKit-Broker] not found` 的校验错误。

地区节点通常分两层：

- `XX自动`：`url-test` 自动测速，适合速度敏感场景。
- `XX节点`：`select` 手动选择，适合账号风控和固定出口场景。

## Shadowrocket 远程规则 URL

完整规则：

```text
https://raw.githubusercontent.com/luzihang123/proxy-rules-kit/main/clients/shadowrocket/routekit.rules
```

精简规则：

```text
https://raw.githubusercontent.com/luzihang123/proxy-rules-kit/main/clients/shadowrocket/routekit-lite.rules
```

这些 URL 只指向公开规则文件，不包含真实节点订阅和 token。

## 公开仓库安全检查

提交前至少执行：

```bash
make check
```

并手动检查以下内容：

- 没有真实订阅 URL 或 token。
- 没有公司域名、内网 IP、家庭网络地址。
- 没有本机绝对路径和账号名称。
- 示例域名使用 `example.com`、`example.test` 等保留域名。
